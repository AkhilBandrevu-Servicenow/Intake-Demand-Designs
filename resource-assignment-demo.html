Demand Workspace · Resource Assignment Demo

Pick a workflow to simulate

Each option is an independent demo of how a new resource assignment is handled. Choose one, then use New to open the creation form and watch the routing behaviour for that option.

Option A

User → Top, else bottom tray

If a Resource (user) is provided, a grouped row with the user name is added to the top tray. If no Resource is given, it drops into the bottom Unassigned assignments tray.

Option B

User → Top tray

If a Resource (user) is provided, a grouped row with the user name is added to the top tray. If no Resource is given, it lands under Unknown (red allocation cells).

Option C

Assigned → Single row table

When a new assignment is created, it appears as a single row in a full role-based table (Image 3 layout) with monthly hours and no bottom tray.
